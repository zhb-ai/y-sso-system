# CLAUDE.md

## Design Context

### 项目概述
SSO 单点登录管理系统 - 企业内部使用的权限和用户管理平台

### 用户画像
- **主要用户**：企业内部管理员、IT管理员、系统管理员
- **使用场景**：管理用户权限、配置系统参数、查看缓存和日志
- **用户目标**：高效完成管理任务，确保系统安全稳定运行

### 品牌个性
**专业、可靠、清新**
- 专业：体现企业级系统的稳定性和权威性
- 可靠：让用户感到信任，操作有明确的反馈
- 清新：简约不复杂，避免过度设计，保持界面清爽

### 视觉方向

#### 风格定位
**清新简约 + 现代商务**
- 大量留白，内容为王
- 清晰的视觉层次
- 减少不必要的装饰元素
- 避免AI设计风格的过度装饰

#### 参考案例
- **Vercel / Linear**：极简现代，深色主题，适合开发者工具
- 强调功能性，减少视觉噪音

#### 色彩策略
- **简化颜色使用**：一个页面只有1-2个突出主题色
- 其他使用默认色值，避免过多色彩干扰
- **主色调**：蓝绿色 `#3B82F6` (RGB: 59, 130, 246) - 体现科技感和专业性
- **中性色**：完整的灰度系统 gray-50 到 gray-900
- **功能色**：success(绿 #22C55E)、warning(黄 #FBBF24)、danger(红 #EF4444)
- **颜色计算**：使用 `color-mix()` 替代透明度计算，生成主色系的浅色/深色变体

#### 可访问性
- 基础可访问性支持
- 确保足够的对比度
- 基本的键盘导航支持

### 设计原则

1. **功能优先**
   - 不修改功能，只做样式调整
   - 保持原有交互逻辑

2. **克制用色**
   - 限制颜色数量
   - 使用中性色为主，主题色点缀

3. **清晰层次**
   - 通过间距和字号建立视觉层次
   - 重要信息突出，次要信息弱化

4. **一致性**
   - 保持组件风格统一
   - 遵循 Element Plus 设计规范

5. **效率导向**
   - 减少用户认知负担
   - 操作路径清晰直观

6. **清新简约**
   - 移除多余的渐变和阴影效果
   - 使用纯色和微妙的过渡效果
   - 保持界面清爽不油腻
   - 避免AI设计风格的过度装饰和复杂效果

### 技术栈
- Vue 3 + Vite
- Element Plus 组件库
- Pinia 状态管理
- CSS 变量主题系统

## 开发规范

### 样式规范
- 使用 CSS 变量而非硬编码值
- 不修改组件自带样式
- 不使用内联样式
- 使用标准 CSS 类和 scoped 样式
- 优先使用 `var(--el-color-primary)` 引用主色
- 半透明效果仅用于悬停背景等必要场景

### 组件规范
- 遵循 Vue 3 组合式 API
- 组件名使用大驼峰命名
- Props 使用小驼峰命名
