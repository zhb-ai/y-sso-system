# Frontend Design

Create distinctive, production-grade frontend interfaces with high design quality. Use this skill when the user asks to build web components, pages, artifacts, posters, or applications. Generates creative, polished code that avoids generic AI aesthetics.

## Core Principles

### 1. Visual Distinctiveness
- **Avoid AI patterns**: Reject generic grids, rounded cards, identical spacing
- **Strong typography**: Create dramatic hierarchies with unexpected font pairings
- **Purposeful color**: Use color strategically for emphasis, not decoration
- **Intentional whitespace**: Use negative space as a design element

### 2. Production-Ready Code
- **No placeholder images**: Use real, relevant imagery or icons
- **Semantic HTML**: Prioritize accessibility and SEO
- **Performance-first**: Optimize for speed and smooth interactions
- **Responsive by default**: Design for all screen sizes from the start

### 3. Creative Freedom
- **Break conventions**: Challenge typical UI patterns when appropriate
- **Personality-first**: Design with character and emotional resonance
- **Memorable details**: Add subtle, delightful touches that users notice

## Design Guidelines

### Typography
- Create clear hierarchies using contrast in size, weight, and spacing
- Use a maximum of 2-3 complementary font families
- Prioritize readability while maintaining visual interest
- Use line-height and letter-spacing strategically

### Color
- Develop a cohesive palette with 1-2 primary colors and supporting accents
- Use color to guide user attention and indicate state
- Ensure sufficient contrast for accessibility
- Consider color psychology and emotional impact

### Layout
- Use grid systems as a foundation, but break them intentionally
- Create visual rhythm through repetition and variation
- Prioritize content hierarchy in spatial arrangement
- Balance symmetry and asymmetry for dynamic compositions

### Interaction
- Design meaningful micro-interactions that provide feedback
- Animate with purpose—every animation should communicate something
- Consider gesture and touch for mobile experiences
- Ensure keyboard accessibility for all interactions

## Implementation Steps

1. **Understand Requirements**: Clarify goals, audience, and constraints
2. **Conceptual Design**: Explore visual directions and creative approaches
3. **Design System**: Establish typography, color, and component patterns
4. **Build Interface**: Create responsive, accessible components and pages
5. **Refine Details**: Polish interactions, animations, and visual touches
6. **Test & Iterate**: Validate on real devices and gather feedback

## When to Use

- Building new web pages or applications
- Creating reusable UI components
- Designing landing pages or marketing artifacts
- Redesigning existing interfaces for better user experience
- Creating visual prototypes or proofs of concept

## What to Avoid

- Generic, template-based designs
- Overuse of gradients, shadows, and animations
- Ignoring accessibility guidelines
- Sacrificing usability for visual flair
- Inconsistent design patterns
