# Y-SSO 设计上下文

## Design Context

### 项目概述
Y-SSO 是一个单点登录（SSO）管理系统，基于 Vue 3 + Vite + Element Plus 构建，用于管理企业组织架构、员工账号、应用权限等。

### 技术栈
- **框架**: Vue 3.5+ with `<script setup>`
- **构建工具**: Vite (rolldown-vite@7.2.5)
- **UI 库**: Element Plus 2.12+
- **状态管理**: Pinia 3.0+
- **图标**: @element-plus/icons-vue
- **样式**: CSS 变量 + 公共样式文件

### 用户
- **主要用户**: 企业内部管理员和普通员工
- **使用场景**: 
  - 管理员：管理组织架构、员工账号、应用权限、SSO配置
  - 员工：使用SSO登录各个业务系统
- **用户目标**: 快速、安全地完成身份认证和权限管理任务

### 品牌个性
- **核心理念**: 简洁高效
- **品牌关键词**: 专业、可靠、清晰、高效
- **情感目标**: 
  - 给用户信心：系统稳定可靠
  - 提升效率：界面清晰，操作流畅
  - 降低认知负担：直观的交互设计

### 视觉方向
- **风格**: 简约现代
- **主题**: 保留项目现有颜色值（以青绿色为主色调）
- **特点**:
  - 干净、留白充足
  - 信息层次清晰
  - 现代化但不张扬
  - 专注功能而非装饰

### 设计原则

1. **简洁至上**
   - 每个元素都应有明确的目的
   - 避免不必要的装饰
   - 保持界面清爽，留白充足

2. **一致性**
   - 使用统一的CSS变量
   - 遵循BEM命名规范
   - 保持组件样式和行为一致

3. **可维护性**
   - 样式优先使用公共CSS文件
   - 页面组件尽量不使用scoped样式
   - 使用CSS变量而非硬编码值

4. **可用性**
   - 清晰的视觉层次
   - 直观的操作流程
   - 适当的反馈和提示

5. **性能优先**
   - 避免过度动画
   - 优化加载性能
   - 保持代码简洁

### 代码规范

#### 导入顺序
1. Vue内置API
2. 第三方库
3. 内部组件
4. 工具函数和API
5. 状态管理

#### 路径规范
- 必须使用 `@/` 路径别名
- 禁止使用相对路径 `../`

#### 样式规范
- 优先使用CSS变量
- 遵循BEM命名规范
- 使用 `:deep()` 进行Element Plus样式穿透
- 禁止直接修改 `.el-xxx` 类名

### 文件结构
```

├── components/       # 组件
│   ├── common/       # 通用组件

│   └── business/     # 业务组件

├── styles/           # 样式文件
│   ├── base/         # 基础样式
│   ├── components/   # 组件样式
│   ├── layout/       # 布局样式
│   ├── pages/        # 页面样式
│   └── utilities/    # 工具类
├── utils/            # 工具函数
└── stores/           # 状态管理
```

### 注意事项
- ❌ 禁止在页面组件中添加大量CSS样式
- ❌ 禁止使用内联样式（除动态样式外）
- ❌ 禁止使用硬编码颜色值、尺寸值
- ❌ 禁止使用 `!important`
- ❌ 禁止直接修改 `.el-xxx` 类名
- ✅ 必须使用CSS变量
- ✅ 必须使用 `@/` 路径别名
- ✅ 必须使用语义化HTML标签
